Optimizing a Simple Regression Model in Theano

Posted Mar 2, 2016 2016-03-02T07:00:00+02:00 by S. Aya

Prior to this post, we discussed neural network design and implementation in Julia language. There are many resources on artificial neural networks. I learn by reading, however, it was overwhelming to find the right book/paper for the first steps. I choose to start with studying the foundations and implementing what I learned to my favourite language. In the meantime, I saw a lot references to deep learning methods in Kaggle forums, Strata conferences and upcoming O’Reilly books. Not to mention journal papers. Thanks to open source community, abstract notions are getting reachable, user-friendly tools: Theano and Keras libraries are the topic of this and upcoming post, respectively. As defined in its home page, Theano is a Python library that allows you to define, optimize, and evaluate mathematical expressions involving multi-dimensional arrays efficiently. Theano is not a deep learning library. It is beyond deep learning. In this post, we investigate the inner workings of symbolic manipulation and evaluation of mathematical expressions using Theano.

Problem Definition

The dataset comes from the scikit-learn datasets. The diabetes dataset consists of 10 physiological variables (age, sex, weight, blood pressure) measure on 442 patients, and an indication of disease progression after one year. The dataset in scikit-learn is standarized (zero mean and unit L2 norm). All right, our focus is on Theano. So, we use one feature and see how the target variable changes with this feature.

Theano Model

As a general convention, capital letters are used for symbolic variables, and lower letters are used for numpy arrays.

X, Y = T.vectors('input', 'desired')
W = theano.shared(.01)  # Shared because they should live in memory (either in CPU or GPU)
B = theano.shared(0.)  # The values inside paranthesis are the values for initializing shared variables.
Z = W*X + B  # This is output (symbolic variable)
Cost = ((Y-Z)**2).mean()  # Cost is the mean squared value
params = [W, B]  # Parameters that should be learned

W.get_value()  # we move the shared variable from Theano to a regular python array

See the result after typing W in the console: <TensorType(float64, scalar)>. This means that W is a theano shared variable. To change the W theano shared variable, we have to defne a theano function and pass it to extra updates list. This list contains shared variable new value pairs. In our example we want the new value of W to be 2W.

2

updates = ((W, W*2),  )
double_w = function([], [], updates=updates)

double_w()  # Call it.
W.get_value()  # Get the value.

Training the model: We use this idea to construct a single layer learning model. We calculate the gradients of the cost function with respect to the parameters using the rule the new value of the previous parameter is just that value minus.

\[\text{C}=(\text{true}-\text{estimated})^2\] \[w=w-\frac{\eta dC}{dw}=2 \eta x (\text{true}-\text{estimated})+w\] \[b=w-\frac{\eta dC}{db}=b+2 \eta (\text{true}-\text{estimated})\]

lr = .1  # Learning rate
grads = [T.grad(Cost, p) for p in params]  # Take derivatives
updates = [(p, p - lr*g) for p,g in zip(params, grads)]  # Update weight and bias
print(updates)

train = function([X, Y], Cost, updates=updates)

Take-aways

This algorithm we just trained optimized with mean squared error is actually one of the first adaptive methods. All other neural methods follow the same idea, however, using the deep models instead of linear ones and possibly different cost functions instead of linear ones.
