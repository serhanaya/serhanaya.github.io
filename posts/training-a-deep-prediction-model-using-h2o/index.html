Posted May 1, 2016 2016-05-01T08:00:00+03:00 by S. Aya
Updated May 3, 2020 2020-05-03T18:07:36+03:00

The objective of this study is to train and build a deep prediction model. The focus is on feedforward neural networks. We will start with the exploration of data, try to simplify it and after that, we will apply our model and investigate the results. In these steps we will use R language. For the modeling process we will use H2O R package. The data comes from a recently ended Kaggle competition: Santander Customer Satisfaction.

The H2O package

H2O implements almost all common machine learning algorithms — such as generalized linear modeling (linear regression, logistic regression, etc.), Naive Bayes, principal components analysis, time series, k-means clustering and other important algorithms such as Random Forest, Gradient Boosting and Deep Learning at scale. H2O is written in Java and is fast and scalable. The H2O R package provides an interface to the H2O software.

Modeling

Feedforward neural networks are useful for prediction and classification where there is a clearly defined outcome (what digit an image contains in digit recognition, whether someone is walking upstairs or walking on a flat surface in activity analyses, the presence/absence of disease in medical studies, and so on). In these cases, there is no particular need for a feedback loop. Recurrent networks are useful for cases where feedback loops are important, such as for natural language processing.

# Start to H2O instance. Our configuration:
# Memory used: 12 GB; Number of CPUs used: 5.
cl <- h2o.init(

    nthreads = 5)

# Import train data frame to the H2O cloud.
h2osan.train <- as.h2o(
    s.train,
    destination_frame = "h2osantrain")

# Import test data frame to the H2O cloud.
h2osan.test <- as.h2o(
    s.test,
    destination_frame = "h2osantest")

In the following model, hyperbolic tangent Tanh is used as activation function. The tanh function is a rescaled and shifted logistic function and its symmetry around 0 allows the training algorithm to converge faster. We use 200 hidden neurons. CrossEntropy is selected as the loss function which has typical use in classification problems. Epochs (passes over the data) per iteration on N nodes is 100. More epochs can be used for greater predictive accuracy, but only when able to afford the computational cost. The training error value is based on the parameter score_training_samples, which specifies the number of randomly sampled training points to be used for scoring; the default uses 10,000 points. The validation error is based on the parameter score_validation_samples, which controls the same value on the validation set and is set by default to be the entire validation set. Setting either of these parameters to 0 automatically uses the entire corresponding dataset for scoring.

Two important metrics for evaluation of the classification model are as follows. AUC, stands for area under the curve (higher is better). Here, the curve is the receiver operating characteristic curve, or ROC curve for short. The ROC curve shows the sensi‐ tivity of the classifier by plotting the rate of true positives to the rate of false positives. In other words, it shows you how many correct positive classifications can be gained as you allow for more and more false positives. The perfect classifier that makes no mistakes would hit a true positive rate of 100% immediately, without incurring any false positives (not possible). Logloss, or logarithmic loss, gets into the finer details of a classifier (lower is better).

    diagnostics = TRUE,
    export_weights_and_biases = TRUE
)

By default the learning rate is 0.005 in h2o.deeplearning() function. Changing it definitely changes the results. For example, taking rate=0.001 changes the result of AUC to 0.831185 (see the value for rate=0.005 below) for training dataset. However for validation dataset the AUC drops to 0.8097628.
