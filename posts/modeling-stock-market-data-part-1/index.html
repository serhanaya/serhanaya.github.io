Modeling Stock Market Data - Part 1

Posted Dec 13, 2015 2015-12-13T07:00:00+02:00 by S. Aya

This is going to be a high level observation of Turkish stock market (BIST) with focus on getting stock fundamentals and then develop a criteria to select good stocks using provided data. Playing the stock market is thrilling. But on this financial roller-coaster ride, we all want to experience the ups without the downs. This observation will hopefully provide an intuitive perspective on the stock markets in general. Although this study is about stock markets, the focus is on the data analysis process: Without using a financial software, can I get financial data? How about using web scraping tools? But, wait, free tools may not give high quality data like the data provided in a commercial financial software: What to do the with all that “?, %, B”s around my numeric values -how to clean data? These questions are the topics of this part of the study. Next, with all that statistics of stocks, we’ll explore using visualization tools and try to create a rule for scoring these stocks. I’ll use Python libraries BeautifulSoup, Requests (scraping), Pandas (manipulation), seaborn and matplotlib (visualization) along the way.

Background

We’ll start by looking at GARAN.IS stock page in Yahoo Finance. Let’s see what we’ve got here: Some basic information on historical prices at the center, a useful chart on the right, news section at lower side, and more importantly a “More on GARAN.IS” section on the left. We’ll deal with historical price data soon, but first, our focus is on Key Statistics. I’ll provide screenshots for important steps. In case Yahoo guys change the layout in the future, for scraping pages one can stick to the principles presented here.

This page provides three main tables: Valuation Measures, Financial Highlights and Trading Information. There are also sub-tables in latter two. We’ll grab these tables using BeautifulSoup library (Python3). First, let’s have a look at the GARAN.IS key statistics page in Figure 1. (1) See web page address. A minor change will soon provide all the information we need about BIST stocks. (2) We’re on Key Statistics page. (3) This is Chrome’s Developer Tools section. Chrome: View \ Developer Tools. We need this tool to scrape the page properly. (4) On Key Statistics page, as an example, right click Valuation Measures table head. Click on Inspect Elements button. You’ll see that some elements in HTML code are highlighted according to the place the cursor on.

Data Retrieval

Tables that we want to acquire are tagged as yfnc_datamodoutline1 (see Figure 1 - blue arrows). Below the title, there are two columns for each table which are tagged as yfnc_tablehead1and yfnc_tabledata1, respectively (see Figure 1 - green arrows). First, we’ll loop through the tables, and for each table we’ll get the head before getting whole data. Thus, in this first step we’ll have the row names. Afterwards, we will need the name of all stocks in BIST. Stay tuned!

from urllib.request import urlopen
from urllib.error import HTTPError
from bs4 import BeautifulSoup
import pandas as pd

x = []

html = urlopen("http://finance.yahoo.com/q/ks?s=GARAN.IS+Key+Statistics")
bsObj = BeautifulSoup(html, "lxml")

for table in bsObj.findAll("table",{"class":"yfnc_datamodoutline1"}):

    for col in table.findAll("td", {"class":"yfnc_tablehead1"}):
        x.append(col.get_text())

fin = pd.DataFrame(x, columns=["KeyStats"])

Well, the code snippet provided here may not be the best, but, here is the explanation of what is going on: urlopen is used to open a remote object across a network and read it. Then a BeautifulSoup object is created to extract data from html file. In the loops, we searched for class tags with yfnc_datamodoutline1 to find the relevant table. Lastly, we get heads and columns from tables and convert acquired data to pandas DataFrame.

In Python 3.x, urllib2 was renamed urllib and was split into several submodules: urllib.request, urllib.parse, and url lib.error. Although function names mostly remain the same, you might want to note which functions have moved to submodules when using the new urllib.

Now we have the row names for what-we-are-about-to-get data using GARAN.IS key statistics page. If you look carefully to tag-1 in Figure 1, changing GARAN.IS to some other stock name with .IS in the end in the address bar will take you to that stock’s key statistics page. That’ great! So if we have the ID’s for each stock in BIST, we can have all the fundamental data we need! Luckily, here is the list of all companies provided by Public Disclosure Platform. I’ve downloaded “All Companies” and “Sectors” and saved as excel files. companies = pd.read_excel(".../data/company_names.xlsx")  # change this to your relative path

inf_stocks = []  # stocks that don't have sufficient information.
err_stocks = []  # stocks that can't be downloaded due to a HTTPError.

for stock in companies["ID"]:
    try:
        html = urlopen("http://finance.yahoo.com/q/ks?s=" "%s" ".IS+Key+Statistics" % stock)
        bsObj = BeautifulSoup(html, "lxml")

        x = []; y = []

        for table in bsObj.findAll("table",{"class":"yfnc_datamodoutline1"}):

            for col in table.findAll("td", {"class":"yfnc_tabledata1"}):
                y.append(col.get_text())

        if y == []:
            inf_stocks.append(stock)
            continue
        else:
            fin[stock] = y
            print(stock)

    except HTTPError as e:
        print("%s: %s" % (stock, e))
        err_stocks.append(stock)
        continue

After opening url with urlopen, due to insufficient information or any other type of HTTP Error some stock information may not be retreived. The HTTP error may be “404 Page Not Found,” “500 Internal Server Error,” etc. In all of these cases, the urlopen function will throw the generic exception HTTPError. We can handle this exception using try/except block. If an HTTP error code is returned, the program now prints the error. The problematic stocks will be listed under inf_stocks and err_stocks lists. During the process you’ll see stock names one by one as output. After the execution, better save acquired financial data to .csv file to be used in further analysis.

Data Cleansing

Time to clean the data! I’ve selected four stocks to see what we are up against: KeyStats column seems a little complicated than it really should be. We should stick to the definitions of the row names in Yahoo page, and try to shorten retreived data as possible.

I’m going to use Regular Expressions here, to select the parts to be deleted (or retained), and hopefully reduce complexity of KeyStats row names. After that, I’m gonna select the stats of each stocks and try to delete % or B or M and convert string to numeric data where possible.

I’ll remove the yellow colored parts of KeyStats column using this Regex object: \s*(\(.+\))?\d?(:). Examining the yellow colored section, one can see that section starts with optional blank \s*, followed by paranthesis and words, spaces whatever there is -we don’t care what’s inside- (\(.+\))?, this section is optional so I put a question mark at the end. After that, some of the rows contain a numeric followed by a semicolon. Numeric character is optional \d?, semicolon is required (:).

import re
keyRegEx = re.compile(r'\s*(\(.+\))?\d?(:)')

for ind, stat in zip(fin.KeyStats.index, fin.KeyStats.values):
    new_stat = stat.replace(keyRegEx.search(stat).group(), '')
    fin.KeyStats.values[ind] = new_stat

For each stock B (for billion), M (for million), K (for thousand) and percent (%) symbols will be removed. Resulting string will be converted to float and multiplied by 1e9 for B, 1e6 for M, 1e3 for K and 1e-2 for %. The data in remaining rows will be converted to float as well, where appropriate.

companies = fin.columns.values[1:]  # company names

for stock in companies:
    for ind, val in zip(fin[stock].index, fin[stock].values):
        try:
            if str(val).endswith('B'):
                val = val[:-1]
                val = float(val) * 1e9
            elif str(val).endswith('M'):
                val = val[:-1]
                val = float(val) * 1e6
            elif str(val).endswith('K'):
                val = val[:-1]
                val = float(val) * 1e3
            elif str(val).endswith('%'):
                val = val[:-1]
                val = float(val) * 0.01
            else:
                val = float(val)
        except:
            None
        fin[stock].values[ind] = val
