Visual Analysis of Tax Data Using Mathematica

Posted Jan 1, 2016 2016-01-01T07:00:00+02:00 by S. Aya
Updated May 2, 2020 2020-05-02T17:19:33+03:00

This study is first appeared on Practical Data Science Book by Tony Ojeda. In this post, Mathematica is used instead of Python on the analysis of the same dataset. I want to show the flexibility of Mathematica at the stages of the data analysis pipeline. Mathematica’s built-in interpreter provides additional information for a given entity and visualization capabilities help creating beautiful graphical presentations. Besides these great properties, Wolfram Mathematica is constantly changing while still maintaining full compatibility with previous versions and has a great user community. The dataset we explore was created by Alvaredo, Facundo, Anthony B. Atkinson, Thomas Piketty, and Emmanuel Saez, The World Top Incomes Database. It contains global information about the highest incomes per country for approximately the past 100 years, gleaned from tax records.

Analysis

[In1] Importing data. It’s pretty straightforward. The data can be taken either from the link provided above or from Packt’s page for the book. [In2] Since the first list of the imported data is column number, we can skip it using the Rest function. It will take all the data except the first one. Also, to see the dimensions of this dataset we use Dimensions built-in function. There are 2180 observations and 354 features. Unfortunately, the dataset contains many missing or invalid entries, which makes sense given the historical data, and countries that may not have had effective data collection for given columns. [In3 & 4] The column names are not explicitly shown here, but the first ten of these names can be quickly checked with the following command: headers = temp[[1]][[;;10]]. First column is the Country of observation. Second column is the Year of observation. In order to use Mathematica’s Date functions for plotting etc. we should convert the head of the expressions in the second column to DateObject. However, first, we need to convert the Head of second column from Integer to String. One can observe the Head of a sample row by using Map[Head, table[[rowname]], {2}]. Here, {2} is the level specification, and the number 2 is suitable for this table. [In5] Testing the procedure. What’s the type (head) of the “Year” column? [In6 & 7] /@ stands for Map. After these steps, the second column of income table, which consists of Year of the observations, has DateObject head.

[In1] temp = Import[".../data/income_dist.csv"]

[In2] (income = Rest[temp])//Dimensions

[Out2] {2180, 354}

[In3] tempDate = ToString/@income[[;;,2]];

[In4] income[[;;,2]] = tempDate;

[In5] income[[;;,2]][[1]] // Head

[In6] tempDate2 = DateObject /@ tempDate

[In7] income[[;; , 2]] = tempDate2;

A quick inspection reveals that although I wrote Turkey as the country name in the previous line, the listed properties are not specific to given country. A total of 749 properties exists, which may or may not be available for given country.

There are 354 features (columns) which starts with Country Name followed by Year. In order to select a specific country, we’ll use the following line. We want to get data related to United States. Here, BlankSequence (__) means the rest of the columns will be taken without any restriction. SQL-like function GroupBy provides a generalization of the map reduce operation. The first parameter of this function is a list, the second, that will be grouped together (in this case, the last part of the list Average Income, will be grouped according to the first part of the list—by Year), and the third parameter—a function that will be applied to the grouped data, that is, in this case its addition.

incomeUSA = Cases[income, {"United States", __}]

GroupBy[incomeUSA[[All, {2, 281}]], First -> Last, Total]

The average income in U.S. Dollars is given at column 283. Let’s check how this value changed in the last century using DateListPlot command. We can add more details to the code, but you’ll see that it is pretty readable with current configuration. For example, no need to adjust the ticks in time axis. The layout is automatically adjusted using the PlotTheme function which was introduced in Mathematica 10, and it is perfect. The important point is to have DateObject time values and corresponding output given in the argument list.

Fig. 4 shows that after World War II, U.S. average income increased steadily. The sharp increase continued until 1970s. After a stall period in those years, average income continue to rise until today. The data for the United States is broken up into six groups as (1) Top 10 percent income share (2), Top 5 percent income share (3), Top 1 percent income share (4), Top 0.5 percent income share, (5) Top 0.1 percent income share (6) Average income share. After examining which column is related with these categories, it is easy to plot different graphs related to the questions in mind. For example, in Figure 5, U.S. percentage income for different groups. Raw percentages for the income groups tend to move in the same direction. When one group’s income increases, the other groups’ incomes also increase. There is also a clear, persistent difference between each of the lines. Using similar pattern, we can generate another plot, as given in Figure 6. We simply choose a different collection of columns and then customize our chart accordingly. This graph shows average incomes by year of the different groups. From the 1980s forward, the difference between wealthy and the lower income groups has increased dramatically.

In this last example, we use built-in GeoRegionValuePlot function to visually show the results of Income data for a specific year. Then, we use Manipulate function to add interactivity to the results. In the code snippet below, the main function is the user-defined plotMap function. To understand the contents, please look at the help file for GeoRegionValuePlot function.

 countryIncome[countryName_?StringQ] := Cases[income, {countryName, __}]

 countryIncomeByYear[data_?ListQ] :=
  GroupBy[data[[All, {2, 283}]], First -> Last, Total]

 dataByYear[data_?ListQ, year_?IntegerQ] :=
  countryIncomeByYear[data][DateObject[ToString[year]]]

 assoc[year_?IntegerQ] :=
  Interpreter["Country"][#] -> dataByYear[countryIncome[#], year] &

 plotMap[year_?IntegerQ, cnames_?ListQ] := GeoRegionValuePlot[
   assoc[year] /@ cnames, ImageSize -> Large,
   PlotLabel ->
    "Average income " <> "for selected countries in " <> ToString@year ,
   ImageSize -> 1000]

Manipulate[
   plotMap[year, {"United States", "France", "Germany", "Italy",
     "South Africa"}], {year, 1975, 2005, 5}]

Take-aways

  • Mathematica is great for small-sized data. Built-in functions are extensive, the system contains almost all-known algorithms for data analysis. As the result, user feels complete control over data, at least that’s what I felt. By the way, visualization tools are the best!
  • This sophisticated tool lags behind with the size of the data.

Recommended Reading

  • Wolfram, S. (2015) “An Elementary Introduction to the Wolfram Language”, Wolfram Media Inc. A quick introduction to the Wolfram Language, will also be helpful for understanding more advanced techniques in functional programming and symbolic languages in general.

  • Wagner, D.B. (1996) “Power Programming With Mathematica: The Kernel (Programming Tools for Scientists & Engineers)”, Mcgraw-Hill. This book never gets old. Very important for learning the details of programming in Mathematica. It covers procedural programming, functional programming and rule based programming in great detail.

  • Ojeda, T. (2014) “Practical Data Science Cookbook - Real-World Data Science Projects to Help You Get Your Hands On Your Data”, Packt Publishing. Cookbook style data science case studies using R and Python. I occasionally refer to this book and try to expand the presented analyses.
